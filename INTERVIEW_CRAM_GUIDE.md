# Interview Cram Guide & Question Database

This guide is designed to prepare you for technical interviews in 1 to 2 hours. It covers the essentials of React, Next.js, Three.js, and your specific AI Engineer portfolio project, followed by a comprehensive, structured question database.

---

## Part 1: Technology & Project Essentials

### React Essentials Cheat Sheet
- **Modularity**: Code is written in reusable **components** (functions returning JSX).
- **One-Way Data Flow**: Data flows down from parent to child via read-only **props**. Updates are requested by triggering callbacks passed down as props.
- **State Management**: Use `useState` for local component values that trigger a redraw when updated.
- **Effects Lifecycle**: Use `useEffect` for operations that run after render (e.g., event listeners, fetching). Remember to return a cleanup function to avoid memory leaks.
- **Key Reference Rule**: Never update React state directly (`state = newValue`). Always use the setter function (`setState`).

### Next.js Essentials Cheat Sheet
- **Framework over Library**: Next.js is a React framework that handles routing, server rendering, and builds.
- **App Router**: Uses file-system routing starting at `app/`. A folder defines a route segment, and a `page.tsx` file inside it renders that route.
- **Server Components (RSC)**: React components that render on the server. They are the default in Next.js, reducing JavaScript sent to the browser.
- **Client Components**: Marked with `"use client"` at the top. They are hydrated in the browser to support state, hooks, and event handlers.
- **Hydration**: The process of taking a static HTML page generated by the server and attaching event listeners/JavaScript state in the browser.

### Three.js & R3F (React Three Fiber) Essentials Cheat Sheet
- **Core Scene Graph**: Three.js consists of a **Scene** (container), a **Camera** (viewpoint), and a **Renderer** (renders scene to canvas).
- **Meshes**: A visible object composed of a **Geometry** (shape) and a **Material** (appearance).
- **Lighting**: Essential for materials to reflect light (e.g., AmbientLight for general light, DirectionalLight for shadows/highlights).
- **R3F**: A React wrapper for Three.js. Standard Three.js JS commands are written declaratively as JSX tags (e.g., `<mesh>`, `<boxGeometry>`).
- **Drei**: A helper library containing pre-made components for R3F (e.g., `<OrbitControls />`, `<Html />` wrappers).
